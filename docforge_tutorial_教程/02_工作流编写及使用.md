# DocForge 教程 02：工作流编写及使用

本教程介绍如何编写和使用工作流。工作流是 DocForge 的核心，它定义了数据处理步骤的顺序和配置。

## 1. 工作流概述

工作流是一个 JSON 文件，它描述了一系列处理步骤。每个步骤调用一个插件，并传递相应的配置参数。工作流按步骤顺序执行，上一个步骤的输出作为下一个步骤的输入。

## 2. 工作流 JSON 结构

### 2.1 基本结构
```json
{
  "name": "工作流名称",
  "description": "工作流描述",
  "version": "1.0.0",
  "steps": [
    {
      "step_id": "步骤ID",
      "plugin_name": "插件名称",
      "config": {
        // 插件配置参数
      }
    }
  ]
}
```

### 2.2 字段说明
- `name`（必需）：工作流名称，用于标识工作流。
- `description`（可选）：工作流描述。
- `version`（可选）：工作流版本，默认为 "1.0.0"。
- `steps`（必需）：步骤数组，按顺序执行。

### 2.3 步骤字段
- `step_id`（必需）：步骤唯一标识，用于调试和日志。
- `plugin_name`（必需）：要调用的插件名称，必须与插件注册的名称一致。
- `config`（可选）：插件配置对象，参数取决于具体插件。

## 3. 官方插件及其配置

### 3.1 excel_extractor（Excel 数据提取器）
从 Excel 文件提取数据。

**配置参数**：
```json
{
  "PlaceHolders": [
    {
      "PlaceHolderName": "字段名称",
      "KeepEmptyCells": "False",
      "Sheet": ["Sheet1"],
      "Column": ["col:B", "colname:公司名称"],
      "Row": [""],
      "Range": ["B2:C5"]
    }
  ]
}
```

**参数说明**：
- `PlaceHolderName`（必需）：字段名称，在后续步骤中使用 `{字段名称}` 引用。
- `KeepEmptyCells`（可选）：是否保留空单元格，默认为 "False"。
- `Sheet`（可选）：指定工作表，空值或 `[""]` 读取所有工作表。
- `Column`（可选）：指定列，格式为 `["col:B"]`（按列号）或 `["colname:公司名称"]`（按列标题）。
- `Row`（可选）：指定行，格式为 `["row:2"]`（按行号）或 `["rowname:标题"]`（按行标题）。
- `Range`（可选）：指定区域，如 `["B2:C5"]`。

**示例**：
```json
{
  "step_id": "extract",
  "plugin_name": "excel_extractor",
  "config": {
    "PlaceHolders": [
      {
        "PlaceHolderName": "公司名称",
        "Sheet": ["Sheet1"],
        "Column": ["colname:公司名称"]
      },
      {
        "PlaceHolderName": "职位",
        "Sheet": ["Sheet1"],
        "Column": ["colname:职位"]
      }
    ]
  }
}
```

### 3.2 docx_replacer（Word 替换器）
将数据填入 Word 模板，生成新文档。

**配置参数**：
```json
{
  "Target": ["template1.docx", "template2.docx"],
  "Ignore": ["temp.docx"],
  "KeepPlaceHolderIfShorter": false,
  "ReplacerOfShorterPlaceHolder": ""
}
```

**参数说明**：
- `Target`（可选）：正选列表，指定要处理的模板文件，支持通配符。空值或 `[""]` 选择所有模板。
- `Ignore`（可选）：反选列表，忽略的文件。优先级高于 Target。
- `KeepPlaceHolderIfShorter`（可选）：当替换值比占位符短时是否保留占位符，默认为 false。
- `ReplacerOfShorterPlaceHolder`（可选）：当 `KeepPlaceHolderIfShorter` 为 true 时，用于替换的字符。

**示例**：
```json
{
  "step_id": "replace",
  "plugin_name": "docx_replacer",
  "config": {
    "Target": ["审批单.docx"],
    "Ignore": ["临时*.docx"]
  }
}
```

### 3.3 placeholder_splitter（占位符分割器）
将一个字段分割为多个新字段。

**配置参数**：
```json
{
  "Target": "字段名称",
  "Digits": "2",
  "SeparateCharacter": ".",
  "KeepOriginal": "True"
}
```

**参数说明**：
- `Target`（必需）：要分割的目标字段名。
- `Digits`（可选）：数字宽度，用于补零。如 "2" 表示 01, 02, 03...。未指定则自动计算。
- `SeparateCharacter`（可选）：分隔符，默认为 "."。
- `KeepOriginal`（可选）：是否保留原字段，默认为 "True"。

**示例**：
```json
{
  "step_id": "split",
  "plugin_name": "placeholder_splitter",
  "config": {
    "Target": "example",
    "Digits": "2",
    "SeparateCharacter": ".",
    "KeepOriginal": "True"
  }
}
```

### 3.4 date_string_formatter（日期格式化器）
格式化日期字符串。

**配置参数**：
```json
{
  "Target": "日期字段",
  "InputFormat": "%Y-%m-%d",
  "OutputFormat": "%Y年%m月%d日"
}
```

**参数说明**：
- `Target`（必需）：要格式化的字段名。
- `InputFormat`（可选）：输入日期格式，默认自动检测。
- `OutputFormat`（可选）：输出日期格式。

### 3.5 number_string_formatter（数字格式化器）
格式化数字字符串。

**配置参数**：
```json
{
  "Target": "数字字段",
  "DecimalPlaces": 2,
  "ThousandSeparator": true
}
```

**参数说明**：
- `Target`（必需）：要格式化的字段名。
- `DecimalPlaces`（可选）：小数位数。
- `ThousandSeparator`（可选）：是否添加千位分隔符。

## 5. 工作流编写步骤

### 5.1 确定处理流程
1. 输入是什么？（Excel、Word、CSV 等）
2. 需要提取哪些数据？
3. 数据需要如何处理？（分割、格式化、过滤等）
4. 输出是什么？（Word、PDF 等）
5. 模板是什么？

### 5.2 选择插件
根据处理流程选择合适的插件：
- 数据提取：excel_extractor、csv_extractor 等
- 数据转换：placeholder_splitter、date_string_formatter 等
- 文件生成：docx_replacer、pdf_replacer 等

### 5.3 编写步骤
1. 为每个步骤定义唯一的 `step_id`。
2. 指定 `plugin_name`。
3. 根据插件文档编写 `config`。

### 5.4 验证工作流
使用 CoreAPI 的 `validate_workflow` 方法验证工作流定义是否正确。

## 6. 使用工作流

### 6.1 在实例中加载工作流
```python
from docforge.api.core_api import CoreAPI

api = CoreAPI()
api.initialize()

# 加载工作流
api.load_workflow("workflow.json")

# 执行工作流
result = api.execute_workflow("工作流名称")

api.shutdown()
```

### 6.2 动态创建工作流
```python
from docforge.types import WorkflowDefinition, WorkflowStep

# 创建工作流定义
workflow = WorkflowDefinition(
    name="动态工作流",
    description="动态创建的工作流",
    steps=[
        WorkflowStep(
            step_id="extract",
            plugin_name="excel_extractor",
            config={"PlaceHolders": [...]}
        ),
        WorkflowStep(
            step_id="replace",
            plugin_name="docx_replacer",
            config={"Target": ["template.docx"]}
        )
    ]
)

# 保存工作流
import json
with open("dynamic_workflow.json", "w", encoding="utf-8") as f:
    json.dump(workflow.__dict__, f, ensure_ascii=False, indent=2)
```

## 7. 工作流调试

### 7.1 使用日志
在步骤配置中添加 `log` 参数，启用详细日志：
```json
{
  "step_id": "extract",
  "plugin_name": "excel_extractor",
  "config": {...},
  "log": {
    "enabled": true,
    "level": "DEBUG"
  }
}
```

### 7.2 单步执行
可以编写脚本单独执行某个步骤：
```python
from docforge.api.core_api import CoreAPI

api = CoreAPI()
api.initialize()

# 加载插件
api.reload_plugins()

# 获取插件实例
plugin = api.plugin_manager.get_plugin("excel_extractor")

# 执行插件
result = plugin.execute(
    input_files=["data.xlsx"],
    config={"PlaceHolders": [...]}
)

print(result)

api.shutdown()
```

### 7.3 验证数据流
在每个步骤后检查数据：
```python
# 在步骤之间添加验证
result1 = api.execute_workflow("步骤1")
print("步骤1数据:", result1.data)

# 手动调用下一个步骤
plugin2 = api.plugin_manager.get_plugin("placeholder_splitter")
result2 = plugin2.execute(data=result1.data)
print("步骤2数据:", result2.data)
```

## 8. 最佳实践

1. **步骤命名**：使用有意义的 `step_id`，如 "extract_company_names"、"format_dates"。
2. **错误处理**：在工作流中添加验证步骤，检查数据完整性。
3. **版本控制**：为工作流指定版本号，便于管理。
4. **模块化**：将复杂工作流拆分为多个子工作流。
5. **文档**：在 `description` 中详细说明工作流的作用和用法。

## 9. 常见问题

### 9.1 工作流执行失败
检查：
- 插件名称是否正确
- 配置参数是否符合插件要求
- 输入文件是否存在
- 模板文件是否存在

### 9.2 数据不匹配
检查：
- 字段名称是否一致
- 数据长度是否一致（特别是使用分割器后）
- 空值处理是否正确

### 9.3 输出文件为空
检查：
- 模板中是否包含正确的占位符
- 数据是否为空
- 输出目录是否有写入权限
