## 2. 核心概念总结

### 2.1 架构层次
```
用户界面层 (GUI) → 应用服务层 (API) → 核心业务层 → 插件扩展层 → 数据持久层
```

### 2.2 关键组件
1. **CoreAPI**：主入口，封装所有核心功能
2. **工作流引擎**：执行步骤调度和数据传递
3. **插件系统**：动态加载和执行插件
4. **项目管理**：配置和状态管理
5. **文件管理**：文件操作和监视

### 2.3 数据流
```
输入文件 → 提取器 → 数据转换 → 替换器 → 输出文件
```

## 3. 快速入门指南

### 3.1 最小可行示例
```python
from docforge.api.core_api import CoreAPI

# 1. 初始化
api = CoreAPI()
api.initialize()

# 2. 配置
api.create_project("项目", "./project")
api.add_input_files(["data.xlsx"])
api.add_template_files(["template.docx"])
api.set_output_directory("./output")

# 3. 执行
api.load_workflow("workflow.json")
result = api.execute_workflow("工作流名称")

# 4. 结果处理
if result.success:
    print(f"生成 {len(result.output_files)} 个文件")

# 5. 清理
api.shutdown()
```

### 3.2 最小工作流
```json

      "config": {...}

  ]
```

## 5. 常见问题解答

### 5.1 安装和环境
**Q: 如何安装 DocForge？**
A: 克隆仓库，安装依赖：`pip install pandas openpyxl python-docx odfpy`

**Q: 支持哪些 Python 版本？**
A: Python 3.8+，推荐 3.10+

### 5.2 使用问题
**Q: 工作流执行失败怎么办？**
A: 1) 检查日志 2) 验证输入文件 3) 测试单个插件 4) 检查配置语法

**Q: 如何处理大文件？**
A: 使用流式处理，分块读取，增加内存限制

**Q: 插件加载失败？**
A: 检查语法、依赖、基类继承、方法实现

### 5.3 扩展开发
**Q: 如何添加新功能？**
A: 开发新插件，继承对应基类，实现必要方法

**Q: 如何修改核心行为？**
A: 修改 services/ 层的相应模块
