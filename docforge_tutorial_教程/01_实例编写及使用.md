# DocForge 教程 01：实例编写及使用

本教程介绍如何通过编写 Python 脚本（实例）来使用 DocForge 框架完成文档处理任务。

## 1. 实例概述

实例是一个 Python 脚本，它使用 DocForge 的 CoreAPI 来执行工作流。实例的作用是：
- 初始化框架
- 配置项目（输入文件、模板、输出目录）
- 加载工作流
- 执行工作流
- 处理结果

## 2. 基本步骤

### 2.1 导入 CoreAPI
```python
from docforge.api.core_api import CoreAPI
```

### 2.2 初始化框架
```python
api = CoreAPI()
if not api.initialize():
    print("框架初始化失败")
    exit(1)
```

### 2.3 创建或打开项目
```python
# 创建新项目
api.create_project("项目名称", "./项目路径")

# 或打开现有项目
api.open_project("./项目路径/project.json")
```

### 2.4 添加输入文件
```python
input_files = ["data1.xlsx", "data2.xlsx"]
api.add_input_files(input_files)
```

### 2.5 添加模板文件
```python
template_files = ["template1.docx", "template2.docx"]
api.add_template_files(template_files)
```

### 2.6 设置输出目录
```python
api.set_output_directory("./output")
```

### 2.7 加载工作流
```python
workflow_path = "workflow.json"
if not api.load_workflow(workflow_path):
    print("加载工作流失败")
    api.shutdown()
    exit(1)
```

### 2.8 执行工作流
```python
def progress_callback(progress, message):
    print(f"[{progress*100:.1f}%] {message}")

result = api.execute_workflow(
    workflow_name="工作流名称",
    progress_callback=progress_callback
)
```

### 2.9 处理结果
```python
if result.success:
    print(f"执行成功，生成了 {len(result.output_files)} 个文件")
    for file in result.output_files:
        print(f"  - {file}")
else:
    print("执行失败：")
    for error in result.errors:
        print(f"  - {error}")
```

### 2.10 关闭框架
```python
api.shutdown()
```

## 3. 完整示例

以下是一个完整的实例脚本，它使用 Excel 提取器和 Word 替换器来生成文档：

```python
"""
example_instance.py
示例实例：从 Excel 提取数据并填入 Word 模板
"""

import sys
from pathlib import Path

# 添加项目根目录到 Python 路径（如果需要）
project_root = Path(__file__).parent.parent
sys.path.insert(0, str(project_root))

from docforge.api.core_api import CoreAPI

def main():
    print("=" * 50)
    print("DocForge 实例示例")
    print("=" * 50)
    
    # 1. 初始化
    api = CoreAPI()
    if not api.initialize():
        print("初始化失败")
        return
    
    # 2. 创建项目
    project_dir = "./my_project"
    if not api.create_project("示例项目", project_dir):
        print("创建项目失败")
        api.shutdown()
        return
    
    # 3. 添加输入文件
    input_file = "./data/数据表.xlsx"
    if not Path(input_file).exists():
        print(f"输入文件不存在: {input_file}")
        api.shutdown()
        return
    
    if not api.add_input_files([input_file]):
        print("添加输入文件失败")
        api.shutdown()
        return
    
    # 4. 添加模板文件
    template_dir = "./templates"
    template_files = list(Path(template_dir).glob("*.docx"))
    if not template_files:
        print(f"模板目录中没有 .docx 文件: {template_dir}")
        api.shutdown()
        return
    
    if not api.add_template_files([str(f) for f in template_files]):
        print("添加模板文件失败")
        api.shutdown()
        return
    
    # 5. 设置输出目录
    output_dir = "./output"
    if not api.set_output_directory(output_dir):
        print("设置输出目录失败")
        api.shutdown()
        return
    
    # 6. 加载工作流
    workflow_path = "./workflow.json"
    if not api.load_workflow(workflow_path):
        print("加载工作流失败")
        api.shutdown()
        return
    
    # 7. 执行工作流
    print("\n开始执行工作流...")
    print("-" * 50)
    
    def progress_callback(progress, message):
        print(f"[{progress*100:5.1f}%] {message}")
    
    result = api.execute_workflow(
        workflow_name="文档生成工作流",
        progress_callback=progress_callback
    )
    
    print("-" * 50)
    
    # 8. 处理结果
    if result.success:
        print(f"\n✓ 执行成功")
        print(f"  生成文件数: {len(result.output_files)}")
        if result.output_files:
            print("  输出文件列表:")
            for file in result.output_files:
                print(f"    - {file}")
    else:
        print(f"\n✗ 执行失败")
        if result.errors:
            print("  错误信息:")
            for error in result.errors:
                print(f"    - {error}")
    
    # 9. 关闭框架
    api.shutdown()
    print("\n实例执行完成")

if __name__ == "__main__":
    main()
```

## 5. 实例配置

实例可以通过配置文件或命令行参数来调整行为。以下是一个使用配置文件的示例：

```python
import json
from pathlib import Path

def load_config(config_path):
    with open(config_path, 'r', encoding='utf-8') as f:
        return json.load(f)

def main():
    config = load_config("config.json")
    
    api = CoreAPI()
    api.initialize()
    
    # 使用配置中的路径
    api.create_project(config["project_name"], config["project_dir"])
    api.add_input_files(config["input_files"])
    api.add_template_files(config["template_files"])
    api.set_output_directory(config["output_dir"])
    api.load_workflow(config["workflow_path"])
    
    result = api.execute_workflow(config["workflow_name"])
    
    api.shutdown()
```

## 6. 错误处理

实例应该包含适当的错误处理：

```python
try:
    # 初始化
    api = CoreAPI()
    if not api.initialize():
        raise RuntimeError("框架初始化失败")
    
    # 执行工作流
    result = api.execute_workflow("工作流名称")
    
    if not result.success:
        raise RuntimeError(f"工作流执行失败: {result.errors}")
    
except Exception as e:
    print(f"实例执行出错: {e}")
    # 清理资源
    if 'api' in locals():
        api.shutdown()
    sys.exit(1)
finally:
    if 'api' in locals():
        api.shutdown()
```

## 7. 最佳实践

1. **使用绝对路径**：避免因工作目录变化导致的路径问题。
2. **检查文件存在**：在添加文件前检查是否存在。
3. **日志记录**：将执行过程记录到日志文件。
4. **参数化配置**：将可变参数提取到配置文件或命令行参数。
5. **清理临时文件**：在实例结束时清理临时文件。

## 8. 调试技巧

1. **查看日志**：使用 `api.get_logs()` 获取执行日志。
2. **单步执行**：可以分别执行每个插件来调试。
3. **使用示例数据**：准备小规模的测试数据，加快调试速度。
