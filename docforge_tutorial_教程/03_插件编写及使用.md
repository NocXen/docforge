# DocForge 教程 03：插件编写及使用

本教程介绍如何为 DocForge 编写自定义插件。插件是扩展框架功能的核心机制。

## 1. 插件概述

插件是一个 Python 类，它继承自特定的基类，并实现预定义的方法。插件分为四种类型：
- **提取器（Extractor）**：从文件提取数据
- **转换器（Transformer）**：转换数据结构
- **替换器（Replacer）**：将数据填入模板，生成输出文件
- **后处理器（PostProcessor）**：对输出文件进行后处理

## 2. 插件基类

所有插件必须继承自相应的基类。基类定义了插件的接口规范。

### 2.1 基类列表
| 基类 | 类型 | 用途 |
|------|------|------|
| `BaseExtractor` | 提取器 | 从文件提取数据 |
| `BaseTransformer` | 转换器 | 转换数据结构 |
| `BaseReplacer` | 替换器 | 模板替换，生成输出文件 |
| `BasePostProcessor` | 后处理器 | 后处理输出文件 |

### 2.2 导入基类
```python
from docforge.plugins.base import BaseExtractor, BaseTransformer, BaseReplacer, BasePostProcessor
```

## 3. 插件接口规范

### 3.1 必需属性
每个插件类必须定义以下属性：
- `name`：插件唯一名称（字符串）
- `version`：版本号（字符串，如 "1.0.0"）
- `plugin_type`：插件类型（字符串，对应基类类型）

### 3.2 必需方法
每个插件必须实现 `execute` 方法，以及对应类型的特定方法。

### 3.3 可选方法
- `initialize(context)`：初始化插件，接收 `PluginContext` 对象
- `cleanup()`：清理资源

## 4. 编写提取器插件

### 4.1 接口要求
提取器必须实现：
- `extract(file_path)`：从文件提取数据，返回 `DataDict`
- `get_supported_formats()`：返回支持的格式列表

### 4.2 示例：CSV 提取器
```python
"""
csv_extractor.py
CSV 文件提取器
"""

from typing import Dict, List
from docforge.plugins.base import BaseExtractor
from docforge.types import ExecutionResult, DataDict
import csv

class CSVExtractor(BaseExtractor):
    """CSV 文件提取器"""
    
    @property
    def name(self) -> str:
        return "csv_extractor"
    
    @property
    def version(self) -> str:
        return "1.0.0"
    
    @property
    def plugin_type(self) -> str:
        return "extractor"
    
    def extract(self, file_path: str) -> DataDict:
        """
        从 CSV 文件提取数据
        
        Args:
            file_path: CSV 文件路径
            
        Returns:
            DataDict: 提取的数据 {字段名: [值列表]}
        """
        data = {}
        
        with open(file_path, 'r', encoding='utf-8') as f:
            reader = csv.DictReader(f)
            
            # 初始化字段
            if reader.fieldnames:
                for field in reader.fieldnames:
                    data[field] = []
            
            # 读取数据
            for row in reader:
                for field, value in row.items():
                    data[field].append(value if value else "")
        
        return data
    
    def get_supported_formats(self) -> List[str]:
        return [".csv"]
    
    def execute(self, file_path: str = None, **kwargs) -> ExecutionResult:
        """执行提取"""
        if file_path is None:
            return ExecutionResult(
                success=False,
                errors=["file_path parameter is required"]
            )
        
        try:
            data = self.extract(file_path)
            return ExecutionResult(success=True, data=data)
        except Exception as e:
            return ExecutionResult(success=False, errors=[str(e)])
```

## 5. 编写转换器插件

### 5.1 接口要求
转换器必须实现：
- `transform(data)`：转换数据，返回 `DataDict`
- `get_description()`：返回转换器描述

### 5.2 示例：文本大写转换器
```python
"""
text_uppercase.py
文本大写转换器
"""

from typing import Dict, List
from docforge.plugins.base import BaseTransformer
from docforge.types import ExecutionResult, DataDict

class TextUppercase(BaseTransformer):
    """将文本字段转换为大写"""
    
    @property
    def name(self) -> str:
        return "text_uppercase"
    
    @property
    def version(self) -> str:
        return "1.0.0"
    
    @property
    def plugin_type(self) -> str:
        return "transformer"
    
    def transform(self, data: DataDict) -> DataDict:
        """
        转换数据：将所有文本值转换为大写
        
        Args:
            data: 输入数据
            
        Returns:
            DataDict: 转换后的数据
        """
        result = {}
        
        for field_name, values in data.items():
            # 将每个值转换为大写
            result[field_name] = [str(v).upper() for v in values]
        
        return result
    
    def get_description(self) -> str:
        return "将所有文本字段转换为大写"
    
    def execute(self, data: DataDict = None, **kwargs) -> ExecutionResult:
        """执行转换"""
        if data is None:
            return ExecutionResult(
                success=False,
                errors=["data parameter is required"]
            )
        
        try:
            result = self.transform(data)
            return ExecutionResult(success=True, data=result)
        except Exception as e:
            return ExecutionResult(success=False, errors=[str(e)])
```

## 6. 编写替换器插件

### 6.1 接口要求
替换器必须实现：
- `replace(template_path, data, output_path)`：执行替换，返回布尔值
- `get_supported_templates()`：返回支持的模板格式列表

### 6.2 示例：文本文件替换器
```python
"""
text_replacer.py
文本文件替换器
"""

from typing import Dict, List
from pathlib import Path
from docforge.plugins.base import BaseReplacer
from docforge.types import ExecutionResult
import re

class TextReplacer(BaseReplacer):
    """文本文件替换器"""
    
    @property
    def name(self) -> str:
        return "text_replacer"
    
    @property
    def version(self) -> str:
        return "1.0.0"
    
    @property
    def plugin_type(self) -> str:
        return "replacer"
    
    def replace(self, template_path: str, data: Dict[str, str], output_path: str) -> bool:
        """
        替换文本文件中的字段
        
        Args:
            template_path: 模板文件路径
            data: 替换数据 {字段名: 值}
            output_path: 输出文件路径
            
        Returns:
            bool: 是否成功
        """
        try:
            # 读取模板
            with open(template_path, 'r', encoding='utf-8') as f:
                content = f.read()
            
            # 替换字段
            for field, value in data.items():
                # 支持 {字段} 和 {字段:默认值} 格式
                pattern = r'\{' + re.escape(field) + r'(?::[^}]*)?\}'
                content = re.sub(pattern, str(value), content)
            
            # 确保输出目录存在
            Path(output_path).parent.mkdir(parents=True, exist_ok=True)
            
            # 写入输出文件
            with open(output_path, 'w', encoding='utf-8') as f:
                f.write(content)
            
            return True
            
        except Exception as e:
            print(f"替换失败: {e}")
            return False
    
    def get_supported_templates(self) -> List[str]:
        return [".txt", ".md", ".html", ".xml", ".json"]
    
    def execute(self, template_path: str = None, data: Dict[str, str] = None, 
                output_path: str = None, **kwargs) -> ExecutionResult:
        """执行替换"""
        if template_path is None or data is None or output_path is None:
            return ExecutionResult(
                success=False,
                errors=["template_path, data, and output_path are required"]
            )
        
        try:
            success = self.replace(template_path, data, output_path)
            if success:
                return ExecutionResult(
                    success=True,
                    output_files=[output_path]
                )
            else:
                return ExecutionResult(
                    success=False,
                    errors=["Replace operation failed"]
                )
        except Exception as e:
            return ExecutionResult(success=False, errors=[str(e)])
```

## 7. 编写后处理器插件

### 7.1 接口要求
后处理器必须实现：
- `process(input_files, output_dir)`：处理文件列表，返回输出文件列表

### 7.2 示例：文件合并器
```python
"""
file_merger.py
文件合并后处理器
"""

from typing import List
from pathlib import Path
from docforge.plugins.base import BasePostProcessor
from docforge.types import ExecutionResult

class FileMerger(BasePostProcessor):
    """文件合并器"""
    
    @property
    def name(self) -> str:
        return "file_merger"
    
    @property
    def version(self) -> str:
        return "1.0.0"
    
    @property
    def plugin_type(self) -> str:
        return "post_processor"
    
    def process(self, input_files: List[str], output_dir: str) -> List[str]:
        """
        合并多个文本文件
        
        Args:
            input_files: 输入文件列表
            output_dir: 输出目录
            
        Returns:
            List[str]: 输出文件列表
        """
        output_files = []
        
        # 合并所有文件内容
        merged_content = []
        for file_path in input_files:
            if Path(file_path).exists():
                with open(file_path, 'r', encoding='utf-8') as f:
                    content = f.read()
                    merged_content.append(f"=== {Path(file_path).name} ===\n{content}\n")
        
        # 写入合并文件
        if merged_content:
            output_path = Path(output_dir) / "merged_output.txt"
            with open(output_path, 'w', encoding='utf-8') as f:
                f.write("\n".join(merged_content))
            output_files.append(str(output_path))
        
        return output_files
    
    def execute(self, input_files: List[str] = None, output_dir: str = None, 
                **kwargs) -> ExecutionResult:
        """执行合并"""
        if input_files is None or output_dir is None:
            return ExecutionResult(
                success=False,
                errors=["input_files and output_dir are required"]
            )
        
        try:
            output_files = self.process(input_files, output_dir)
            return ExecutionResult(
                success=True,
                output_files=output_files
            )
        except Exception as e:
            return ExecutionResult(success=False, errors=[str(e)])
```

## 8. 使用 PluginAPI

插件可以通过 `PluginAPI` 访问框架功能。在 `execute` 方法中，可以通过 `kwargs.get("plugin_api")` 获取。

### 8.1 PluginAPI 功能
- 日志记录：`log_info()`, `log_error()` 等
- 文件操作：`read_file()`, `write_file()`
- 数据获取：`get_input_data()`, `set_output_data()`
- 配置读取：`get_config()`, `get_plugin_config()`
- 缓存操作：`cache_get()`, `cache_set()`
- 数据库操作：`db_query()`, `db_insert()`
- 事件系统：`emit_event()`, `subscribe_event()`

### 8.2 使用示例
```python
class MyExtractor(BaseExtractor):
    def execute(self, file_path: str = None, **kwargs) -> ExecutionResult:
        # 获取 PluginAPI
        plugin_api = kwargs.get("plugin_api")
        
        if plugin_api:
            # 记录日志
            plugin_api.log_info(f"开始处理文件: {file_path}")
            
            # 读取配置
            encoding = plugin_api.get_config("encoding", "utf-8")
            
            # 使用缓存
            cache_key = f"file_hash_{file_path}"
            cached = plugin_api.cache_get(cache_key)
            
            if cached:
                plugin_api.log_info("使用缓存数据")
                return ExecutionResult(success=True, data=cached)
        
        # 正常处理...
        data = self.extract(file_path)
        
        if plugin_api:
            # 设置缓存
            plugin_api.cache_set(cache_key, data, ttl=3600)
            
            # 发送事件
            plugin_api.emit_event("file_extracted", {"file": file_path})
        
        return ExecutionResult(success=True, data=data)
```

## 9. 插件配置

### 9.1 配置传递
插件配置通过工作流步骤的 `config` 字段传递，在 `execute` 方法中通过 `kwargs.get("config")` 获取。

### 9.2 配置验证
建议在插件中添加配置验证：
```python
def validate_config(self, config: Dict) -> List[str]:
    """验证配置参数"""
    errors = []
    
    if "required_param" not in config:
        errors.append("缺少必需参数: required_param")
    
    if "numeric_param" in config:
        try:
            float(config["numeric_param"])
        except ValueError:
            errors.append("numeric_param 必须是数字")
    
    return errors

def execute(self, **kwargs) -> ExecutionResult:
    config = kwargs.get("config", {})
    
    # 验证配置
    errors = self.validate_config(config)
    if errors:
        return ExecutionResult(success=False, errors=errors)
    
    # 使用配置...
```

## 10. 插件注册与加载

### 10.1 插件目录
将插件文件放在 `docforge_plugins` 目录下，框架会自动加载。

### 10.2 手动加载
```python
from docforge.api.core_api import CoreAPI

api = CoreAPI()
api.initialize()

# 从文件加载插件
api.plugin_manager.load_plugin("./my_plugins/custom_extractor.py")

# 或从目录加载所有插件
api.plugin_manager.load_all_plugins("./my_plugins")

api.shutdown()
```

## 12. 最佳实践

1. **错误处理**：所有可能出错的代码都应捕获异常，并返回适当的 `ExecutionResult`。
2. **日志记录**：使用 `PluginAPI` 记录关键步骤和错误信息。
3. **配置验证**：在 `execute` 方法开始时验证配置参数。
4. **资源清理**：在 `cleanup` 方法中释放资源（如文件句柄、网络连接）。
5. **文档字符串**：为类和方法添加清晰的文档字符串。
6. **类型注解**：使用类型注解提高代码可读性和可维护性。

## 13. 常见问题

### 13.1 插件加载失败
检查：
- 插件类是否继承了正确的基类
- 是否实现了所有必需的属性和方法
- Python 语法是否正确
- 依赖的包是否已安装

### 13.2 插件执行失败
检查：
- 输入参数是否符合预期
- 文件路径是否正确
- 权限是否足够
- 依赖的外部服务是否可用

### 13.3 插件间数据传递
确保：
- 字段名称一致
- 数据类型兼容
- 数据长度匹配（特别是分割器后）
